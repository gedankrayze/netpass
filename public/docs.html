NetPass API Documentation

Welcome to the NetPass API documentation. NetPass is a secure, scalable authentication service built with modern technologies for enterprise applications.

Base URL: All API endpoints are relative to https://netpass.gedankrayze.com/api

Introduction

NetPass provides a comprehensive authentication and user management solution with the following key features:

  • Secure user registration and login
  • JWT-like token authentication
  • User profile management
  • Custom user preferences storage
  • Password management
  • Session management

Installation

To set up NetPass locally, follow these steps:

# Clone the repository
git clone https://github.com/gedankrayze/netpass.git
cd netpass

# Install dependencies
bun install

# Copy environment variables
cp .env.example .env

# Configure your database connection in .env

# Initialize the database
bun run src/scripts/init-db.ts

# Start the development server
bun run dev

Configuration

NetPass uses environment variables for configuration. Create a .env file in the root directory:

# Database configuration
DATABASE_URL=http://localhost:8529
DATABASE_NAME=netpass
DATABASE_USER=root
DATABASE_PASSWORD=yourpassword

# JWT configuration
JWT_TTL_DAYS=7

# Authentication configuration
ALLOW_SIGNUP=true

# Server configuration
PORT=3000

Quick Start

Here's a quick example of how to register a user and authenticate:

// Register a new user
const response = await fetch('https://netpass.gedankrayze.com/api/auth/signup', {
    method: 'POST',
    headers: {
        'Content-Type': 'application/json'
    },
    body: JSON.stringify({
        email: 'user@example.com',
        username: 'johndoe',
        password: 'SecurePass123',
        firstName: 'John',
        lastName: 'Doe'
    })
});

const { user, token } = await response.json();

// Use the token for authenticated requests
const profile = await fetch('https://netpass.gedankrayze.com/api/users/profile', {
    headers: {
        'Authorization': `Bearer ${token}`
    }
});

const userData = await profile.json();

User Registration

POST /api/auth/signup

Register a new user account.

Registration can be disabled by setting ALLOW_SIGNUP=false in environment variables.

Request Body

{
    "email": "user@example.com",
    "username": "johndoe",
    "password": "SecurePass123",
    "firstName": "John",
    "lastName": "Doe"
}

Password Requirements

  • Minimum 8 characters
  • At least one uppercase letter
  • At least one lowercase letter
  • At least one number

Response

{
    "user": {
        "_key": "12345",
        "email": "user@example.com",
        "username": "johndoe",
        "profile": {
            "firstName": "John",
            "lastName": "Doe"
        },
        "custom": {},
        "isActive": true,
        "emailVerified": false,
        "createdAt": "2025-01-25T10:00:00Z",
        "updatedAt": "2025-01-25T10:00:00Z"
    },
    "token": "eyJ0eXAiOiJKV1QiLCJhbGc..."
}
201 Created 400 Bad Request 403 Forbidden (signup disabled) 409 Conflict (email/username taken)

Token Management

NetPass uses secure token-based authentication. Tokens are:

  • Generated using cryptographically secure random bytes
  • Valid for a configurable duration (default: 7 days)
  • Stored securely in the database with user association
  • Validated on each request to protected endpoints
Token expiration can be configured using the JWT_TTL_DAYS environment variable.

Custom Preferences

NetPass allows storing custom user preferences in a flexible JSON format. This can be used for:

  • UI preferences (theme, language)
  • Application-specific settings
  • Feature flags
  • Notification preferences
// Update custom preferences
const response = await fetch('/api/users/profile', {
    method: 'PATCH',
    headers: {
        'Authorization': `Bearer ${token}`,
        'Content-Type': 'application/json'
    },
    body: JSON.stringify({
        custom: {
            theme: 'dark',
            language: 'en',
            notifications: {
                email: true,
                push: false,
                sms: true
            },
            features: {
                betaAccess: true,
                advancedMode: false
            }
        }
    })
});

CORS (Cross-Origin Resource Sharing)

NetPass includes built-in CORS support for cross-origin requests from web applications.

Configuration

CORS can be configured using the CORS_ORIGINS environment variable:

  • CORS_ORIGINS=* - Allow all origins (default)
  • CORS_ORIGINS=http://localhost:3000 - Allow single origin
  • CORS_ORIGINS=http://localhost:3000,https://myapp.com - Allow multiple origins

React Integration Example

// In your React application
const login = async (username, password) => {
    const response = await fetch('https://netpass.gedankrayze.com/api/auth/login', {
        method: 'POST',
        headers: {
            'Content-Type': 'application/json'
        },
        body: JSON.stringify({ username, password })
    });
    
    if (response.ok) {
        const { user, token } = await response.json();
        // Store token for authenticated requests
        localStorage.setItem('token', token);
        return user;
    }
    
    throw new Error('Login failed');
};

// Making authenticated requests
const getProfile = async () => {
    const token = localStorage.getItem('token');
    const response = await fetch('https://netpass.gedankrayze.com/api/users/profile', {
        headers: {
            'Authorization': `Bearer ${token}`
        }
    });
    
    return response.json();
};

Security Considerations

For production deployments:

  • Set CORS_ORIGINS to specific allowed domains instead of using *
  • Use HTTPS for all API endpoints
  • Consider implementing additional security headers

Security Best Practices

  • HTTPS Only: Always use HTTPS in production
  • Strong Passwords: Enforce password requirements
  • Token Storage: Store tokens securely on the client
  • Rate Limiting: Implement rate limiting for authentication endpoints
  • Input Validation: All inputs are validated using Zod schemas
  • Password Hashing: Passwords are hashed using bcrypt with a cost factor of 10
  • CORS Configuration: Restrict allowed origins in production

Error Handling

NetPass uses consistent error responses across all endpoints:

{
    "error": "Descriptive error message",
    "statusCode": 400
}

For validation errors:

{
    "message": "Validation failed",
    "errors": {
        "email": ["Invalid email format"],
        "password": [
            "Password must be at least 8 characters",
            "Password must contain at least one uppercase letter"
        ]
    }
}
